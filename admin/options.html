WP development logger 2.0: options

How to use

Click on Developer > Developer logger.

This will open a second tab, so the php logger will keep working if your development tab crashes.

The logger will capture all 15 php errors and report them. There are three color codes:

  • Entries in RED mean that your script crashed before completion
  • Entries in BLUE mean your script completed but there are warnings
  • Entries in GREEN are your logger entries

Use the function writelog() to record your own logging activities.

You can log any php type - string, integer, array, object etc eg

  • writelog('code is ok up to here');
  • writelog(45);
  • writelog(array('name'=>'bill', 'age'=>45);
  • writelog($var);

MySQL

This logger will also capture the last error emitted by MySQL.

It uses an obscure variable called $EZSQL_ERROR

When your script(s) finish running the plugin checks whether WordPress has populated this. If so, it gets reported

human-readable log

In addition to generating a log that shows up on the logger browser tab, this plugin generates a human-readable text file at your_wordpress_directory/wp-content/log.txt

Depending on how advanced your understanding of logging is, you can also track reporting in log.txt as follows. Assuming you can open a console window and navigate to 'your_wp_install_directory/wp_content' :

  • Windows: c:>\your_wordpress_directory\wp_content> more +1 log.txt
  • Mac: /your_wordpress_directory/wp_content> tail -F log.txt
  • Linux: /your_wordpress_directory/wp_content> tail -f log.txt
